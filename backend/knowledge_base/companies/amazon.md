# Amazon — Campus Placement Profile (LPU)

## Company Overview

Amazon is one of the most coveted recruiters at LPU, visiting annually for campus placements. The company hires primarily for its **India Development Center (IDC)** in Bangalore, Hyderabad, and Chennai. Amazon's hiring bar is known to be extremely high — candidates must demonstrate strong DSA skills, system design thinking, and alignment with Amazon's **Leadership Principles**.

## Recruitment Process — Full Pipeline

### Stage 0: Pre-screening
Amazon shortlists from the placement portal based on CGPA and resume. Students with a strong competitive programming profile (Codeforces, CodeChef, LeetCode) get an edge.

---

### Stage 1: Online Assessment (OA)
- **Platform:** HackerRank
- **Duration:** 2 hours
- **Format:** 2 DSA problems (typically 1 medium + 1 medium-hard, or 1 medium + 1 hard)
- **Additional sections:** Work sample test / situational judgment (sometimes included for SDE-1)
- **Proctored:** Yes — webcam + browser lockdown
- **Cutoff:** Generally need 100% test cases to be safe; 80%+ is borderline

**Typical OA Problem Types:**
- Array manipulation with constraints
- Graph BFS/DFS (implicit graphs)
- DP problems (Knapsack variants, LCS)
- String manipulation (anagram detection, compression)
- Sliding window / two pointer

**OA Tips:**
- Read problem constraints carefully — they hint at the expected time complexity
- Don't just code the brute force; Amazon OA expects optimized solutions
- Edge cases matter — empty arrays, single elements, INT_MAX values
- Write clean code with proper variable names even if under time pressure

---

### Stage 2: Technical Round 1 (TR-1)
- **Duration:** 45–60 minutes
- **Focus:** DSA + Basics of System Design
- **Format:** 1-2 DSA problems + conceptual discussion

**Commonly Asked DSA Topics:**
- Linked List operations (Reverse in K-groups, detect cycle, merge sorted lists)
- Trees (LCA, diameter, max path sum, zigzag traversal)
- Arrays (Subarray problems, Two Sum variants)
- Strings (Longest substring without repeat, valid parentheses)

**System Design Basics (often asked even for SDE-1):**
- What is a REST API?
- Explain client-server architecture
- What is caching? When would you use it?
- Basics of Load Balancing

**Behavioral component:**
- 1-2 Leadership Principle (LP) questions
- Typical: "Tell me about a time you had to handle a tight deadline" (maps to Deliver Results)

---

### Stage 3: Technical Round 2 (TR-2)
- **Duration:** 45–60 minutes
- **Focus:** DSA (harder) + Low-Level Design (LLD)

**DSA Expected Level:**
- Dynamic Programming (medium-hard: Burst Balloons, Edit Distance, Jump Game II)
- Graph algorithms (Shortest Path, Topological Sort)
- Advanced data structures (LRU Cache, Trie, Segment Tree basics)

**LLD Topics:**
- Design a Parking Lot (OOP: classes, relationships, interfaces)
- Design an ATM system
- Design a Library Management System
- Design patterns: Singleton, Factory, Observer

**Leadership Principle Discussion:**
- Usually 2 LP questions this round
- "Describe a time you disagreed with your team and how you handled it" (Disagree and Commit)
- "Give an example of when you went above and beyond" (Customer Obsession)

---

### Stage 4: Bar Raiser Round
- **Duration:** 60–90 minutes
- **Conducted by:** Amazon Bar Raiser (a specially trained interviewer, not from your hiring team)
- **Purpose:** Assess if candidate raises the overall hiring bar
- **Cannot be overridden** by hiring manager — this round is the final filter

**What Bar Raiser Tests:**
- Technical depth (may revisit complex DSA or system design)
- Cultural fit (Amazon LP alignment — all 16 principles)
- Situational judgment and decision-making
- Communication and problem-solving approach

**Typical Bar Raiser Questions:**
- "Walk me through the most complex project you've built" (deep technical dive)
- "Tell me about a failure and what you learned" (Learn and Be Curious)
- "Describe a situation where data drove your decision" (Dive Deep)

---

### Stage 5: HR Round
- Salary negotiation (modest scope as CTC is largely fixed for campus)
- Preferred joining locations (Bangalore, Hyderabad, Chennai)
- Joining timeline / notice period
- Relocation assistance discussion
- Bond clarification (Amazon India generally does not have a bond)

## Interview Tips — Specifically for Amazon

1. **Think Aloud:** Amazon interviewers specifically want to see your thought process. Narrate while solving. Silence is a red flag.

2. **Start with Brute Force:** Always state the naive approach first, analyze its complexity, then optimize. This shows structured thinking.

3. **Draw Diagrams:** For system design and tree/graph problems, ask if you can use a virtual whiteboard. Visual aids help.

4. **STAR Format for Behavioral:** Every LP story must have:
   - **S**ituation: Brief context
   - **T**ask: What was your responsibility
   - **A**ction: What YOU specifically did (use "I", not "we")
   - **R**esult: Quantifiable outcome

5. **Code Quality Matters:** Write clean, readable code. Name variables clearly. Add comments on complex logic.

6. **Edge Cases:** Always ask clarifying questions — what if the array is empty? Can there be duplicates? Are values always positive?

7. **Complexity Analysis:** After every solution, volunteer the time and space complexity without being asked.

8. **Don't Freeze:** If stuck, share your approach verbally. Interviewers can give hints — but you must attempt first.
