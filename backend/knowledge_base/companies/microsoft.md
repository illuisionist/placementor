# Microsoft — Campus Placement Profile (LPU)

## Company Overview

Microsoft is the **highest CTC payer** at LPU, with offers reaching up to **44 LPA** for SDE-1 roles. Microsoft India hires for its **India Development Center (IDC)** locations in Hyderabad, Bangalore, and Noida. The company is known for a rigorous but fair hiring process with deep emphasis on **Object-Oriented Programming, project depth, and system design**.

Microsoft's culture is centered on **Growth Mindset** (Carol Dweck's model) — interviewers specifically look for candidates who demonstrate curiosity, adaptability, and willingness to learn.

## Recruitment Process

### Stage 0: Profile Shortlisting
- Microsoft shares eligibility criteria with LPU placement cell
- Candidates are shortlisted based on CGPA, internships, competitive programming rank, and notable projects
- Resume is the most critical filter — make sure it is achievement-oriented, not duty-oriented

### Stage 1: Online Coding Test
- **Platform:** Codility (or Microsoft's internal platform)
- **Duration:** 90 minutes
- **Format:** 2 DSA problems (medium difficulty)
- **Proctored:** Yes

**Typical Problem Types:**
- Array manipulation (rotate, find missing elements, subarray problems)
- String processing (palindrome, anagram, pattern matching)
- Tree traversal (serialization, path problems)
- Basic DP (subset sum, staircase problem)
- Graph BFS/DFS (connected components, shortest path)

**Cutoff:** Usually need 100% on at least 1 problem and partial credit on the other.

---

### Stage 2: Technical Interview 1 (DSA Focus)
- **Duration:** 45–60 minutes
- **Format:** 1 coding problem + detailed DSA discussion

**Commonly Asked Topics:**
- Linked List: Reverse, detect cycle, find middle
- Trees: Level order BFS, diameter, LCA
- Stacks: Valid parentheses, next greater element
- Arrays: Kadane's algorithm, two sum, three sum
- Sorting: QuickSort implementation, Merge Sort analysis

**Discussion Questions:**
- Time and space complexity analysis
- Best case vs worst case vs average case
- How would you modify your solution for streaming data?
- What if memory is limited?

---

### Stage 3: Technical Interview 2 (OOP + Projects)
- **Duration:** 45–60 minutes
- **Format:** OOP concepts + full project walkthrough

**OOP Questions:**
- Explain the four pillars with real-world examples from your project
- Difference between abstract class and interface (with code examples)
- Design patterns: Singleton, Factory, Strategy — explain and implement
- SOLID principles with violations and fixes
- When would you use composition over inheritance?
- Explain polymorphism and give an example from your code

**Project Discussion — Critical:**
Microsoft expects you to know **every line of code in your projects**. Common questions:
- Walk me through the architecture of your project
- What design patterns did you use? Why?
- What would you change if you had to scale this to 1 million users?
- What was the most challenging bug you faced and how did you debug it?
- Explain the database schema and your choice of database
- Why did you choose [Framework X] over [Framework Y]?
- How did you handle authentication/authorization?
- How would you add a new feature [X] to your current project?

> **Pro Tip:** Do NOT list any project on your resume that you cannot fully explain. Microsoft will dive deep into it.

---

### Stage 3/4: Technical Interview 3 (System Design + Deep Dive)
> *This round may be combined with TR-2 or given separately for SDE-1.*

**System Design Topics (Basics for SDE-1):**
- Design a URL Shortener (focus on: hashing, DB schema, caching)
- Design a Notification Service (pub-sub model, queues)
- Design a chat application (WebSockets, message storage)
- Explain REST vs GraphQL
- Explain how a CDN works
- What is a microservices architecture vs monolith? Trade-offs?

---

### Stage 4/5: HR Round
- Tell me about yourself (practice a crisp 2-minute version)
- Why Microsoft? (Research Microsoft's products and vision — mention specific products)
- Where do you see yourself in 5 years?
- Are you comfortable relocating to Hyderabad/Bangalore?
- Questions about team preferences (product teams vs cloud vs gaming)

## Interview Tips — Specifically for Microsoft

1. **Know your projects inside out.** This is non-negotiable. Microsoft's interviewers will spend 20-30 minutes on your projects alone.

2. **OOP is king.** Microsoft values clean, maintainable code. Brush up on all four pillars with real examples from your own projects.

3. **Design Patterns matter.** Know Singleton, Factory, Observer, Strategy, Decorator, and Builder patterns — with code-level implementation.

4. **Growth Mindset.** If you don't know something, say "I haven't worked with that, but here's how I'd approach it." Don't bluff.

5. **Code readability.** Write clean code with proper indentation, meaningful variable names, and structured logic. Interviewers care about code quality.

6. **Ask clarifying questions.** Before coding, clarify inputs, constraints, and expected output. This is expected behavior.

7. **Practice on Codility.** The OA platform has a different feel from LeetCode — practice timed problems on codility.com.

8. **Research Microsoft's products.** Know about Azure, Microsoft 365, Teams, GitHub (acquired by Microsoft), Xbox Game Studios, and Bing AI. Mention specific products in your "Why Microsoft?" answer.
